This should be easy for telling the difference between languages such as Java and C but may be difficult between very similar languages(C and C++).

School Projects

Mock Shell

This was one of my favorite projects in my OS class. We basically designed a shell which took whatever the user put as an input, parsed the input, forked, change the child to whatever command the user put, do the process,and then get reaped. It was mostly used to teach us about how child processes are created and killed. We also allowed commands to become a background process, had piping and added some of our own commands. I thought the most interesting part was the background process part since that had us use flags to know when to reap the dead processes.

Mock PollEV

This was another project done in my OS class. For this project we made a basic PollEV polling server in C. I liked this project a lot because it introduced us to threading and mutex locks which was something I was intrested in. We had a few different mutex lock types which ranged from fine grain to reader/writer mutexes. As for the multithreading part, it felt easy except for terminating threads which I had gotten a little confused with while writing. The server/client structure we used involved establishing a connection to the client, then creating a user profile(if they were a unique user) and logging them in. Sending the data from the polls to the user was one of the harder parts of the project in my opinion because it involved a lot of resizing the memory of the string dynamically. It also allowed me to appreciate how hard making something multithreaded can be.

Traveling Salesperson Using Heuristics

Traveling salesperson was a cool unsolved problem we were given. We had to write four different algorithms to find total shortest path possible with the fourth one being whatever you wanted it to be. We wrote this in Java and each algorithm had its own principle to follow(i.e. greedy). This was a peer programming project we did in high school.

AI Minesweeper

Code Interpreter Project

In python, we had to make our own code interpreter for a language our professor made up. The language was simple(no spaces in between each line of code) and we had to approach each line in our own way. Overall this project was one of my favorites just because we created our own coding language which functioned in the python terminal.
